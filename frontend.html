HTML/CSS 基础

什么是盒模型？标准盒模型和IE盒模型有什么区别？
高频

盒模型是CSS中用于描述元素占用空间的模型，包括：content（内容）、padding（内边距）、border（边框）、margin（外边距）。

区别：

  • 标准盒模型（content-box）：width/height只包含content
  • IE盒模型（border-box）：width/height包含content + padding + border

使用 box-sizing: border-box 可以统一使用IE盒模型，便于布局计算。

CSS选择器的优先级是如何计算的？
高频

优先级从高到低：

  1. !important：最高优先级，慎用
  2. 行内样式：style属性，权重1000
  3. ID选择器：#id，权重100
  4. 类/属性/伪类选择器：.class、:hover，权重10
  5. 标签/伪元素选择器：div、::before，权重1
  6. 通配符：*，权重0
如何实现水平垂直居中？
高频

方法一：Flex布局（推荐）

.parent {
    display: flex;
    justify-content: center;
    align-items: center;
}

方法二：绝对定位 + transform

.child {
    position: absolute;
    top: 50%;
    left: 50%;
    transform: translate(-50%, -50%);
}
BFC是什么？如何创建BFC？

BFC（Block Formatting Context）是块级格式化上下文，是独立的渲染区域。

创建方式：float不为none、position为absolute/fixed、display为flex/grid、overflow不为visible

作用：清除浮动、防止margin重叠、阻止元素被浮动元素覆盖

JavaScript 核心

JavaScript的数据类型有哪些？如何判断数据类型？
高频

基本类型（7种）：Number、String、Boolean、Null、Undefined、Symbol、BigInt

引用类型：Object（包括Array、Function、Date等）

判断方法：

  • typeof：适合基本类型，null返回"object"
  • instanceof：判断原型链，适合引用类型
  • Object.prototype.toString.call()：最准确，返回"[object Type]"
== 和 === 的区别是什么？
高频
  • ==：宽松相等，会进行类型转换后比较
  • ===：严格相等，类型和值都必须相同
0 == '0'     // true（类型转换后相等）
0 === '0'    // false（类型不同）
null == undefined  // true
null === undefined // false

建议始终使用 === 进行比较，避免隐式类型转换带来的问题。

什么是闭包？闭包有什么作用？
高频

闭包是指有权访问另一个函数作用域中变量的函数，即使那个函数已经执行完毕。

function outer() {
    let count = 0;
    return function inner() {
        count++;
        return count;
    };
}
const counter = outer();
console.log(counter()); // 1
console.log(counter()); // 2

作用：

  • 数据私有化，创建私有变量
  • 实现模块化
  • 函数柯里化
  • 防抖节流函数的实现

注意：闭包会导致变量常驻内存，使用不当可能造成内存泄漏。

解释JavaScript的原型链
高频

每个对象都有一个__proto__属性指向其构造函数的原型对象（prototype），原型对象也有__proto__，形成链条直到null。

function Person(name) {
    this.name = name;
}
Person.prototype.sayHello = function() {
    console.log('Hello, ' + this.name);
};

const p = new Person('Tom');
p.sayHello(); // Hello, Tom

// 原型链
p.__proto__ === Person.prototype
Person.prototype.__proto__ === Object.prototype
Object.prototype.__proto__ === null
this的指向有哪些情况？
高频

this的指向取决于函数的调用方式：

  1. 普通函数调用：this指向window（严格模式为undefined）
  2. 对象方法调用：this指向调用该方法的对象
  3. 构造函数调用：this指向新创建的实例对象
  4. call/apply/bind：this指向绑定的对象
  5. 箭头函数：this继承自外层作用域，无法通过call/apply改变
宏任务和微任务的区别？Event Loop的执行顺序？
高频

宏任务（macrotask）：setTimeout、setInterval、I/O、UI渲染

微任务（microtask）：Promise.then、MutationObserver、queueMicrotask

执行顺序：

  1. 执行同步代码
  2. 执行所有微任务
  3. 执行一个宏任务
  4. 重复步骤2-3
console.log('1');
setTimeout(() => console.log('2'), 0);
Promise.resolve().then(() => console.log('3'));
console.log('4');
// 输出：1 4 3 2
什么是防抖和节流？如何实现？
高频

防抖（debounce）：事件触发后等待一段时间，如果期间再次触发则重新计时。适用于搜索框输入。

function debounce(fn, delay) {
    let timer = null;
    return function(...args) {
        clearTimeout(timer);
        timer = setTimeout(() => fn.apply(this, args), delay);
    };
}

节流（throttle）：一段时间内只执行一次。适用于滚动加载、resize。

function throttle(fn, limit) {
    let inThrottle;
    return function(...args) {
        if (!inThrottle) {
            fn.apply(this, args);
            inThrottle = true;
            setTimeout(() => inThrottle = false, limit);
        }
    };
}

React Hooks有哪些？useEffect的依赖项有什么作用？
高频

常用Hooks：

  • useState：状态管理
  • useEffect：副作用处理
  • useContext：上下文
  • useRef：获取DOM或保存可变值
  • useMemo：缓存计算结果
  • useCallback：缓存函数
  • useReducer：复杂状态管理

useEffect依赖项：

  • 空数组[]：只在组件挂载和卸载时执行
  • 有依赖项：依赖变化时执行
  • 不写依赖项：每次渲染都执行
React的虚拟DOM是什么？diff算法原理？
高频

虚拟DOM是真实DOM的JavaScript对象表示，通过JS对象模拟DOM结构。

优点：

  • 减少直接操作DOM的次数
  • 跨平台（React Native）
  • 批量更新，提高性能

Diff算法（三大策略）：

  1. tree diff：只对同级比较，跨层级移动节点会销毁重建
  2. component diff：不同类型组件直接替换，同类型比较属性
  3. element diff：通过key识别移动、新增、删除
React的setState是同步还是异步的？
高频

setState通常是异步的，但在setTimeout、Promise.then、原生事件中是同步的。

// 异步示例
this.setState({ count: this.state.count + 1 });
console.log(this.state.count); // 旧值

// 同步示例
setTimeout(() => {
    this.setState({ count: this.state.count + 1 });
    console.log(this.state.count); // 新值
}, 0);

批量更新：React会将多个setState合并为一次更新，提高性能。

Vue

Vue的响应式原理是什么？
高频

Vue2：使用Object.defineProperty()对对象的属性进行劫持，在getter中收集依赖，setter中触发更新。

Vue3：使用Proxy代理整个对象，可以监听更多操作（新增属性、删除属性等）。

// Vue3 Proxy示例
const reactive = (target) => {
    return new Proxy(target, {
        get(target, key, receiver) {
            track(target, key); // 收集依赖
            return Reflect.get(target, key, receiver);
        },
        set(target, key, value, receiver) {
            const result = Reflect.set(target, key, value, receiver);
            trigger(target, key); // 触发更新
            return result;
        }
    });
};
Vue的生命周期有哪些？
高频

Vue2生命周期：

  • 创建前/后：beforeCreate / created
  • 挂载前/后：beforeMount / mounted
  • 更新前/后：beforeUpdate / updated
  • 销毁前/后：beforeDestroy / destroyed

Vue3生命周期（组合式API）：

  • setup() 替代 beforeCreate/created
  • onBeforeMount / onMounted
  • onBeforeUpdate / onUpdated
  • onBeforeUnmount / onUnmounted
v-if和v-show的区别？
高频
特性 v-if v-show
渲染方式 条件为false时不渲染 始终渲染，通过display切换
切换开销 高（销毁/重建组件） 低（仅CSS切换）
适用场景 条件很少改变 频繁切换显示
computed和watch的区别？
高频
  • computed：计算属性，有缓存，依赖不变不重新计算，适合简单的数据转换
  • watch：侦听器，无缓存，数据变化时执行回调，适合异步或复杂操作
// computed
const fullName = computed(() => {
    return firstName.value + ' ' + lastName.value;
});

// watch
watch(searchText, (newVal, oldVal) => {
    // 执行搜索请求
    fetchSearchResults(newVal);
});

性能优化

前端性能优化有哪些方法？
重要

加载优化：

  • 代码分割（Code Splitting）、懒加载
  • 资源压缩（Gzip、Brotli）
  • CDN加速、HTTP缓存
  • 图片优化（WebP、懒加载、响应式图片）
  • Tree Shaking删除无用代码

渲染优化：

  • 减少DOM操作，使用文档片段
  • 避免强制同步布局（Forced Synchronous Layout）
  • 使用requestAnimationFrame做动画
  • 虚拟列表处理大量数据
  • CSS动画优先（使用transform、opacity）

JavaScript优化：

  • 防抖节流
  • Web Worker处理复杂计算
  • 事件委托
首屏加载优化有哪些方案？
高频
  1. SSR/SSG：服务端渲染或静态生成，减少首屏白屏时间
  2. 骨架屏：提升用户感知性能
  3. 资源预加载：preload、prefetch关键资源
  4. 按需加载：路由懒加载、组件懒加载
  5. 关键CSS内联：将首屏CSS直接嵌入HTML
  6. 服务端推送：HTTP/2 Server Push

浏览器原理

从输入URL到页面显示发生了什么？
重要
  1. DNS解析：将域名解析为IP地址
  2. TCP连接：三次握手建立连接
  3. 发送HTTP请求：构建请求报文发送给服务器
  4. 服务器处理：服务器处理请求返回响应
  5. 接收响应：浏览器接收HTML、CSS、JS等资源
  6. 解析HTML：构建DOM树
  7. 解析CSS：构建CSSOM树
  8. 合并渲染树：DOM + CSSOM = Render Tree
  9. 布局（Layout）：计算元素位置和大小
  10. 绘制（Paint）：将渲染树绘制到屏幕
  11. 合成（Composite）：将图层合成为最终页面
什么是重绘（Repaint）和重排（Reflow）？如何优化？
高频

重排（Reflow）：当元素尺寸、位置发生变化，浏览器重新计算布局。开销大。

重绘（Repaint）：当元素外观变化但不影响布局（如color、background）。开销较小。

优化方法：

  • 批量修改样式（使用class代替多次style修改）
  • 离线操作DOM（cloneNode、DocumentFragment）
  • 使用transform和opacity（触发GPU加速）
  • 避免频繁读取布局属性（offsetWidth、clientHeight等）
  • 使用will-change或CSS containment
浏览器的缓存策略有哪些？
高频

强缓存（不会发请求到服务器）：

  • Expires：HTTP/1.0，绝对过期时间
  • Cache-Control：HTTP/1.1，max-age=秒数

协商缓存（会发请求验证）：

  • Last-Modified / If-Modified-Since：基于文件修改时间
  • ETag / If-None-Match：基于文件内容哈希，更精确

缓存优先级：强缓存 > 协商缓存 > 重新获取
