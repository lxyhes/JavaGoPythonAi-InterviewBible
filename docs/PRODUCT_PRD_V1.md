# 面试指南产品化改造 PRD v1

## 1. 文档信息
- 产品名称：Mianshi Guide（面试指南）
- 版本：v1.0
- 日期：2026-03-06
- 面向阶段：MVP（4-6 周）

## 2. 背景与问题
当前项目优势是内容覆盖广、分类完整，但核心体验仍偏“知识展示站”。

用户真实痛点：
- 不知道先学什么（时间有限，优先级不清晰）
- 看过就忘，无法形成可复述输出
- 题目分散，缺少学习闭环与进度反馈

产品方向：从“内容库”升级为“面试训练工具”，目标是提升用户面试通过率。

## 3. 产品目标
### 3.1 核心目标
在 4-6 周内完成 MVP，让用户完成“选题 -> 训练 -> 复盘”的闭环。

### 3.2 业务目标（MVP）
- 用户首日完成一次完整训练闭环的比例 >= 35%
- 7 日留存提升（相对当前）>= 20%
- 人均每次访问有效学习题数 >= 8 题

### 3.3 用户目标
- 30 分钟内知道该看什么
- 面试前 7 天有可执行训练计划
- 能快速定位薄弱项并反复练习

## 4. 目标用户与场景
### 4.1 用户画像
- 应届生：需要结构化入门与高频题
- 1-3 年开发者：需要冲刺查漏补缺
- 转岗用户：需要按岗位快速补齐知识

### 4.2 场景
- 面试前 7 天冲刺
- 当天临时复习（碎片时间）
- 模拟面试前自测

## 5. MVP 范围（必须做）
### 5.1 全局搜索与过滤
功能：
- 支持关键词搜索（问题标题、答案文本）
- 支持标签过滤：必问/高频/重要
- 支持分类过滤：前端/后端/数据库/算法/系统设计/DevOps/网络/OS/AI

交互要求：
- 搜索输入后 200ms 内本地返回结果
- 结果点击后跳到具体题目锚点
- 无结果时提供推荐关键词

### 5.2 刷题模式（训练模式）
功能：
- 卡片仅先展示问题，点击“显示答案”展开
- 题目状态：不会 / 模糊 / 会讲
- 快捷键支持（如 1/2/3 标记）

交互要求：
- 标记状态后自动跳下一题（可配置）
- 支持“仅看未掌握题”

### 5.3 复习队列
功能：
- 自动聚合“不会 + 模糊”题目为复习队列
- 支持按分类/标签过滤复习队列
- 显示“今日待复习数”

策略（MVP 版本）：
- 简化遗忘策略：1 天、3 天、7 天复习提醒
- 暂不做复杂算法，先规则驱动

### 5.4 学习路径页
功能：
- 提供预置路径：
  - 前端 7 天冲刺
  - 后端 7 天冲刺
  - Java 后端 30 题
- 每个路径包含：题目列表、预计时长、完成进度

### 5.5 学习看板
功能：
- 今日学习题数
- 已掌握题数
- 当前连续学习天数
- 各分类掌握率

## 6. 非目标（MVP 不做）
- 用户登录与云端同步
- 社区/评论/排行
- AI 面试官实时追问
- 服务端搜索（MVP 使用本地索引）

## 7. 信息架构与页面
- 首页：进入训练路径 + 最近学习
- 搜索页：全局搜索 + 筛选 + 结果列表
- 分类页：题目列表 + 目录导航
- 刷题页：题卡、答案、状态标记、下一题
- 复习页：待复习列表 + 今日计划
- 个人看板页：进度统计

## 8. 关键用户流程
### 8.1 首次使用
首页 -> 选择“7天冲刺路径” -> 开始刷题 -> 标记掌握度 -> 生成复习队列 -> 查看今日进度

### 8.2 回访用户
打开站点 -> 首页看到“今日待复习” -> 进入复习模式 -> 完成队列 -> 看板查看提升

## 9. 数据模型（前端本地存储）
建议使用 Pinia + localStorage。

### 9.1 题目实体
- id
- category
- section
- question
- answer
- tags[]

### 9.2 用户学习记录
- questionId
- mastery（unknown / vague / mastered）
- lastReviewedAt
- reviewCount
- nextReviewAt

### 9.3 会话记录
- date
- solvedCount
- masteredCount
- durationMinutes

## 10. 交互与 UX 规范
- 搜索框固定在顶部，支持键盘快捷打开（Ctrl/Cmd + K）
- 刷题页面主按钮固定位置（显示答案、标记状态）
- 移动端单手操作区域放底部
- 关键行为给即时反馈（标记成功 toast）
- 页面切换和动画保持轻量，优先响应速度

## 11. 指标与埋点
### 11.1 核心指标
- 训练闭环完成率
- 7 日留存
- 人均学习题数
- 复习队列完成率

### 11.2 关键埋点
- search_submit
- question_open
- answer_expand
- mastery_mark
- review_queue_complete
- path_start / path_complete

## 12. 技术方案建议
- 前端：Vue3 + Pinia（沿用）
- 搜索：本地构建轻量索引（可先 simple includes，后续替换 Fuse.js）
- 状态：新增 `learningStore` 管理学习记录与复习队列
- 复习任务：基于本地时间计算 nextReviewAt

## 13. 版本规划
### Sprint 1（第 1 周）
- 全局搜索 + 过滤
- 搜索结果跳题目锚点

### Sprint 2（第 2 周）
- 刷题模式
- 掌握度标记与本地持久化

### Sprint 3（第 3 周）
- 复习队列与规则（1/3/7 天）
- 今日复习入口

### Sprint 4（第 4 周）
- 学习路径页
- 学习看板与基础埋点

## 14. 风险与对策
- 风险：内容结构不统一导致搜索/路径配置复杂
  - 对策：先统一 question schema，建立转换层

- 风险：本地存储量增长影响性能
  - 对策：记录最小化（仅状态和时间戳），按分类懒加载

- 风险：移动端交互拥挤
  - 对策：优先完成“底部操作栏 + 卡片化布局”

## 15. 验收标准（MVP）
- `type-check` 与 `lint` 全通过
- 至少 3 条学习路径可用
- 刷题模式可完整执行（看题/看答案/标记/下一题）
- 复习队列可自动生成并完成
- 看板可展示今日统计与分类掌握率
