MySQL的索引类型有哪些？B+树索引的原理？
高频

索引类型：

  • 主键索引：唯一标识每行记录
  • 唯一索引：列值必须唯一
  • 普通索引：加速查询
  • 全文索引：用于全文搜索
  • 组合索引：多列联合索引

B+树特点：

  • 非叶子节点只存索引，不存数据，可存储更多索引
  • 叶子节点包含所有数据，且通过链表连接，便于范围查询
  • 查询效率稳定，都是O(logN)
什么情况下索引会失效？
高频
  • 使用!=、<>、NOT IN
  • 使用OR连接条件（两边都有索引才会用）
  • 对索引列进行函数操作或计算
  • like以%开头
  • 字符串类型不加引号（隐式类型转换）
  • 组合索引不满足最左前缀原则
事务的ACID特性是什么？隔离级别有哪些？
高频

ACID：

  • Atomicity（原子性）：事务要么全部成功，要么全部失败
  • Consistency（一致性）：事务前后数据完整性约束不被破坏
  • Isolation（隔离性）：事务之间相互隔离
  • Durability（持久性）：事务提交后数据永久保存

隔离级别（从低到高）：

  1. Read Uncommitted：读未提交，可能脏读
  2. Read Committed：读已提交，可能不可重复读
  3. Repeatable Read：可重复读（MySQL默认），可能幻读
  4. Serializable：串行化，最高隔离级别
MVCC是什么？实现原理？
高频

MVCC（多版本并发控制）：通过保存数据的历史版本，实现读写不阻塞，提高并发性能。

实现原理：

  • 隐藏字段：每行记录有DB_TRX_ID（事务ID）和DB_ROLL_PTR（回滚指针）
  • undo log：保存数据修改前的版本，形成版本链
  • ReadView：快照读时生成，判断数据版本可见性
SQL优化的方法有哪些？
重要
  • 避免SELECT *，只查询需要的字段
  • 为WHERE、JOIN、ORDER BY的列建立索引
  • 避免在WHERE中对字段进行函数操作
  • 使用EXPLAIN分析执行计划
  • 避免大事务，分批处理
  • 使用连接（JOIN）代替子查询
  • 使用LIMIT分页，避免深度分页
  • 批量插入使用INSERT INTO ... VALUES (...), (...)
MySQL主从复制的原理？

复制流程：

  1. Master将数据变更记录到binlog（二进制日志）
  2. Slave的IO线程读取Master的binlog，写入relay log（中继日志）
  3. Slave的SQL线程重放relay log，实现数据同步

复制模式：异步复制、半同步复制、组复制

Redis的持久化方式有哪些？
高频

RDB（快照）：

  • 定时将内存数据快照保存到磁盘
  • 文件紧凑，恢复速度快
  • 可能丢失最后一次快照后的数据

AOF（追加文件）：

  • 记录每个写操作命令
  • 数据安全性高，最多丢失1秒数据
  • 文件较大，恢复速度慢

混合持久化（Redis 4.0+）：RDB做全量，AOF做增量

Redis的过期删除策略和内存淘汰策略？
高频

过期删除策略：

  • 定时删除：设置过期时间的同时创建定时器（CPU压力大）
  • 惰性删除：访问时检查是否过期（内存压力大）
  • 定期删除：每隔一段时间随机检查一批key（折中方案）

内存淘汰策略（maxmemory-policy）：

  • volatile-lru：从已设置过期时间的key中淘汰最近最少使用
  • allkeys-lru：从所有key中淘汰最近最少使用
  • volatile-ttl：淘汰即将过期的key
  • volatile-random/allkeys-random：随机淘汰
  • noeviction：不淘汰，直接返回错误（默认）
Redis如何实现分布式锁？
高频

基本实现：

SET key value NX EX 30  # NX：不存在才设置，EX：过期时间

Redisson实现（推荐）：

  • 使用看门狗机制自动续期
  • 支持可重入锁
  • 支持红锁（RedLock）解决主从延迟问题

注意事项：

  • 设置过期时间，防止死锁
  • value设置唯一标识，释放时检查（防止误删）
  • 使用Lua脚本保证原子性
Redis缓存穿透、击穿、雪崩及解决方案？
重要

缓存穿透：查询不存在的数据，直接打到数据库

  • 方案：布隆过滤器、缓存空值

缓存击穿：热点key过期，大量请求打到数据库

  • 方案：互斥锁、逻辑过期（不设置TTL）

缓存雪崩：大量key同时过期，数据库压力激增

  • 方案：随机过期时间、多级缓存、熔断降级
Redis主从复制、哨兵、集群的区别？
  • 主从复制：一主多从，读写分离，手动故障转移
  • 哨兵（Sentinel）：监控主从节点，自动故障转移，提供发现服务
  • 集群（Cluster）：数据分片（16384个slot），支持水平扩展，自动故障转移
