进程与线程

进程和线程的区别？
高频
特性 进程 线程
资源 独立地址空间 共享进程资源
开销 大（创建/切换） 小
通信 IPC（管道、消息队列等） 直接读写共享内存
崩溃影响 不影响其他进程 可能导致整个进程崩溃
进程间通信（IPC）方式有哪些？
高频
  • 管道（Pipe）：半双工，父子进程间通信
  • 命名管道（FIFO）：无亲缘关系进程间通信
  • 消息队列：消息的链表，有格式
  • 共享内存：最快，需配合信号量使用
  • 信号量：用于进程同步
  • 信号：异步通知机制
  • Socket：不同主机间通信
进程的调度算法有哪些？
  • 先来先服务（FCFS）：简单但可能产生长作业等待
  • 短作业优先（SJF）：平均等待时间最短
  • 时间片轮转（RR）：每个进程分配时间片
  • 优先级调度：根据优先级选择
  • 多级反馈队列：结合时间片和优先级
死锁产生的条件？如何避免？
高频

死锁四条件（缺一不可）：

  1. 互斥：资源一次只能被一个进程占用
  2. 占有且等待：占有资源同时等待其他资源
  3. 不可抢占：已分配的资源不能被强制剥夺
  4. 循环等待：进程间形成循环等待链

避免方法：

  • 银行家算法（资源分配前检查安全性）
  • 资源有序分配法（按序申请资源）
  • 超时放弃
  • 一次性申请所有资源

内存管理

虚拟内存是什么？有什么作用？
高频

虚拟内存是一种内存管理技术，使每个进程认为自己拥有连续的内存空间，实际物理内存可能不连续。

作用：

  • 扩大可用内存（使用磁盘作为交换空间）
  • 内存隔离（进程间互不干扰）
  • 简化内存管理（连续的虚拟地址）
  • 实现内存共享（共享库）
页面置换算法有哪些？
  • OPT（最佳置换）：淘汰将来最长时间不使用的（理论最优）
  • FIFO（先进先出）：可能产生Belady异常
  • LRU（最近最少使用）：最常用，基于局部性原理
  • LFU（最少使用）：考虑使用频率
  • Clock（时钟置换）：LRU的近似实现
什么是缺页中断？处理流程？

缺页中断：访问的页面不在物理内存中，需要从磁盘加载。

处理流程：

  1. CPU产生缺页中断，陷入内核
  2. 保存当前进程状态
  3. 查找页面在磁盘的位置
  4. 分配物理页框（可能需要页面置换）
  5. 从磁盘读取页面到内存
  6. 更新页表
  7. 恢复进程状态，重新执行指令

Linux文件权限755、644是什么意思？
高频

三位数字分别代表：所有者、所属组、其他用户的权限

  • 4：读（r）
  • 2：写（w）
  • 1：执行（x）

755 = rwxr-xr-x（所有者可读可写可执行，组和其他只读可执行）

644 = rw-r--r--（所有者可读可写，组和其他只读）

Linux进程状态有哪些？
  • R（Running）：运行中或可运行
  • S（Sleeping）：可中断睡眠（等待事件）
  • D（Disk sleep）：不可中断睡眠（通常IO）
  • T（Stopped）：停止（收到信号）
  • Z（Zombie）：僵尸进程（已终止但父进程未回收）
如何排查CPU占用高的问题？
  1. top或htop查看CPU占用高的进程
  2. top -Hp PID查看进程中占用高的线程
  3. printf "%x\n" TID将线程ID转为16进制
  4. jstack PID | grep TID查看线程堆栈（Java）
  5. 分析代码，定位问题
Linux的IO模型有哪些？
重要
  • 阻塞IO：调用后等待数据准备好
  • 非阻塞IO：立即返回，轮询检查
  • IO多路复用：select/poll/epoll，同时监听多个fd
  • 信号驱动IO：数据准备好后发送信号
  • 异步IO：数据准备好并拷贝到用户空间后通知

epoll优势：基于事件通知，无fd数量限制，O(1)时间复杂度

Shell编程

Shell脚本中$0、$1、$#、$?、$*、$@的区别？
  • $0：脚本名称
  • $1-$9：第1-9个参数
  • $#：参数个数
  • $?：上一个命令的退出状态
  • $*：所有参数（作为一个字符串）
  • $@：所有参数（作为多个字符串）
  • $$：当前进程ID
Shell中单引号和双引号的区别？
  • 单引号''：原样输出，不解析变量和转义字符
  • 双引号""：解析变量和转义字符
name="World"
echo 'Hello $name'  # 输出：Hello $name
echo "Hello $name"  # 输出：Hello World
