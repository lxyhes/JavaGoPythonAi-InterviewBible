Java

Java中的集合类有哪些？List、Set、Map的区别？
高频

Collection接口：

  • List：有序可重复（ArrayList、LinkedList、Vector）
  • Set：无序不可重复（HashSet、TreeSet、LinkedHashSet）

Map接口：

  • 键值对存储（HashMap、TreeMap、LinkedHashMap、ConcurrentHashMap）

区别：List允许重复保持顺序；Set不允许重复；Map存储键值对

HashMap的底层原理是什么？
高频

底层结构：数组 + 链表 + 红黑树（JDK1.8）

JDK1.7 vs 1.8：1.7使用头插法可能产生死循环；1.8使用尾插法，链表长度≥8转为红黑树

put流程：计算hash→计算索引→无冲突直接放入→有冲突用链表/红黑树→超过阈值0.75扩容

synchronized和ReentrantLock的区别？
高频
  • synchronized是JVM层面的，自动获取/释放锁
  • ReentrantLock是API层面的，需要手动lock/unlock
  • ReentrantLock支持可中断、超时获取、公平锁
  • ReentrantLock可以绑定多个Condition
Java线程池的核心参数有哪些？
高频
  1. corePoolSize：核心线程数
  2. maximumPoolSize：最大线程数
  3. keepAliveTime：非核心线程存活时间
  4. workQueue：任务等待队列
  5. threadFactory：线程工厂
  6. handler：拒绝策略

拒绝策略：AbortPolicy（抛异常）、CallerRunsPolicy（调用者执行）、DiscardPolicy（静默丢弃）、DiscardOldestPolicy（丢弃最老）

JVM的内存结构是怎样的？
重要

运行时数据区：

  • 堆（Heap）：存放对象实例，分新生代（Eden、Survivor）和老年代
  • 方法区（Method Area）：存储类信息、常量、静态变量（JDK8后为元空间Metaspace）
  • 虚拟机栈（VM Stack）：方法执行的内存模型，存储局部变量、操作数栈
  • 本地方法栈（Native Stack）：Native方法服务
  • 程序计数器（PC Register）：当前线程执行的字节码行号指示器
垃圾回收算法有哪些？G1垃圾收集器的特点？
高频

垃圾回收算法：

  • 标记-清除：标记无用对象，然后清除。会产生内存碎片
  • 复制算法：将内存分为两块，复制存活对象到另一块。用于新生代
  • 标记-整理：标记后将存活对象向一端移动。用于老年代
  • 分代收集：新生代用复制，老年代用标记-整理/清除

G1收集器特点：

  • 将堆划分为多个Region，优先回收价值最大的Region
  • 可预测的停顿时间模型
  • 整体采用标记-整理，局部采用复制算法
  • 适合大堆内存（6G以上）
Spring的IOC和AOP是什么？
高频

IOC（控制反转）：将对象的创建和管理交给Spring容器，通过依赖注入（DI）实现。降低耦合，提高可测试性。

AOP（面向切面编程）：将横切关注点（日志、事务、权限）从业务逻辑中分离。

AOP实现方式：

  • JDK动态代理（实现接口）
  • CGLIB代理（继承类）
Spring Bean的生命周期？
高频
  1. 实例化（new/反射创建对象）
  2. 属性赋值（populateBean，依赖注入）
  3. 如果实现了BeanNameAware，调用setBeanName
  4. 如果实现了BeanFactoryAware，调用setBeanFactory
  5. 如果实现了ApplicationContextAware，调用setApplicationContext
  6. BeanPostProcessor.postProcessBeforeInitialization
  7. 如果实现了InitializingBean，调用afterPropertiesSet
  8. 调用自定义init-method
  9. BeanPostProcessor.postProcessAfterInitialization
  10. Bean就绪使用
  11. 容器关闭时，如果实现了DisposableBean，调用destroy
  12. 调用自定义destroy-method
Spring Boot的自动配置原理是什么？
高频

核心注解：@SpringBootApplication

  • @Configuration：标识配置类
  • @ComponentScan：组件扫描
  • @EnableAutoConfiguration：开启自动配置

自动配置流程：

  1. 读取META-INF/spring.factories中的自动配置类

Python

Python的GIL是什么？有什么影响？
高频

GIL（Global Interpreter Lock）是Python解释器的全局解释器锁，确保同一时刻只有一个线程执行Python字节码。

影响：

  • 多线程无法利用多核CPU（IO密集型任务除外）
  • CPU密集型任务应使用多进程（multiprocessing）
  • IO密集型任务可以使用多线程或协程

解决方案：多进程、协程（asyncio）、使用C扩展（释放GIL）

Python的装饰器是什么？如何实现？
高频

装饰器是一种设计模式，用于在不修改原函数代码的情况下扩展功能。

# 函数装饰器
def my_decorator(func):
    def wrapper(*args, **kwargs):
        print("Before call")
        result = func(*args, **kwargs)
        print("After call")
        return result
    return wrapper

@my_decorator
def say_hello():
    print("Hello")

# 带参数的装饰器
def repeat(num):
    def decorator(func):
        def wrapper(*args, **kwargs):
            for _ in range(num):
                result = func(*args, **kwargs)
            return result
        return wrapper
    return decorator
Python的生成器和迭代器有什么区别？
高频

迭代器（Iterator）：实现了__iter__()和__next__()的对象

生成器（Generator）：特殊的迭代器，使用yield关键字定义

# 迭代器
class MyIterator:
    def __init__(self, max_num):
        self.max_num = max_num
        self.current = 0
    
    def __iter__(self):
        return self
    
    def __next__(self):
        if self.current < self.max_num:
            num = self.current
            self.current += 1
            return num
        raise StopIteration

# 生成器（更简洁）
def my_generator(max_num):
    current = 0
    while current < max_num:
        yield current
        current += 1

生成器优势：惰性求值，节省内存，适合处理大数据

Python的深拷贝和浅拷贝区别？
高频
  • 浅拷贝（copy.copy）：创建新对象，但只复制引用，内部对象共享
  • 深拷贝（copy.deepcopy）：递归复制所有对象，完全独立
import copy

# 浅拷贝
a = [1, 2, [3, 4]]
b = copy.copy(a)
b[2].append(5)  # a也会变成 [1, 2, [3, 4, 5]]

# 深拷贝
c = copy.deepcopy(a)
c[2].append(6)  # a保持不变
Python的__init__、__new__、__call__区别？
  • __new__：创建实例对象的方法，在__init__之前调用
  • __init__：初始化实例对象的方法
  • __call__：使实例对象可调用，像函数一样
class Singleton:
    _instance = None
    
    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance

class Counter:
    def __call__(self):
        return "Counter called"

Go

Go的goroutine和thread的区别？
高频
  • goroutine：轻量级线程，由Go运行时管理，创建成本极低（约2KB栈空间）
  • thread：操作系统线程，创建成本高（约1MB栈空间）
  • goroutine之间通过channel通信，避免锁竞争
  • goroutine调度由Go运行时完成，而非操作系统
// 启动goroutine
go func() {
    fmt.Println("Hello from goroutine")
}()

// channel
ch := make(chan int)
go func() {
    ch <- 42
}()
value := <-ch
Go的map底层实现？并发安全如何实现？
高频

map底层结构：hmap + bucket数组，每个bucket存8个key-value

并发安全：Go原生的map不是并发安全的，需要使用sync.Map或加锁（sync.RWMutex）

// 并发安全的map
var syncMap sync.Map
syncMap.Store("key", "value")
value, ok := syncMap.Load("key")
syncMap.Delete("key")
Go的defer、panic、recover机制？
高频
  • defer：延迟执行，函数返回前调用，常用于资源释放
  • panic：触发panic，类似异常，会终止程序
  • recover：捕获panic，防止程序终止
func safeCall() {
    defer func() {
        if r := recover(); r != nil {
            fmt.Println("Recovered:", r)
        }
    }()
    panic("Something went wrong")
}
Go的select语句有什么用？

select用于监听多个channel的读写操作，类似于switch，但只用于channel。

select {
case msg := <-ch1:
    fmt.Println("Received from ch1:", msg)
case msg := <-ch2:
    fmt.Println("Received from ch2:", msg)
case <-time.After(time.Second):
    fmt.Println("Timeout")
}

Node.js

Node.js的事件循环机制？
高频

事件循环阶段：

  1. timers：执行setTimeout、setInterval
  2. pending callbacks：执行延迟的I/O回调
  3. idle, prepare：内部使用
  4. poll：获取新的I/O事件，处理除了close、timers、setImmediate外的回调
  5. check：执行setImmediate回调
  6. close callbacks：执行socket close等回调

微任务：Promise.then、process.nextTick（优先于其他微任务）

Node.js的模块加载机制？require和import的区别？
高频
  • require：CommonJS规范，同步加载，运行时解析
  • import：ES6规范，静态导入，编译时解析（可做tree-shaking）
// CommonJS
const fs = require('fs');
module.exports = {};

// ES6
import fs from 'fs';
export default {};
Node.js如何实现高并发？
  • 事件驱动：非阻塞I/O，单线程处理大量并发请求
  • Cluster：多进程利用多核CPU
  • Worker Threads：多线程处理CPU密集型任务
  • Stream：流式处理大数据

服务注册与发现有哪些方案？

分布式事务如何解决？
重要
  • 2PC（两阶段提交）：协调者向参与者发送prepare和commit，存在同步阻塞、单点故障问题
  • 3PC（三阶段提交）：增加preCommit阶段，降低阻塞范围
  • TCC（补偿事务）：Try（预留资源）、Confirm（确认执行）、Cancel（取消预留）
  • 最终一致性：通过消息队列实现，RocketMQ事务消息
  • Seata：开源分布式事务解决方案，支持AT模式
