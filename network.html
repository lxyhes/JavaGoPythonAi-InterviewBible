TCP/IP

OSI七层模型和TCP/IP四层模型的区别？
高频

OSI七层：物理层、数据链路层、网络层、传输层、会话层、表示层、应用层

TCP/IP四层：网络接口层、网络层、传输层、应用层

对应关系：

  • TCP/IP网络接口层 = OSI物理层 + 数据链路层
  • TCP/IP网络层 = OSI网络层
  • TCP/IP传输层 = OSI传输层
  • TCP/IP应用层 = OSI会话层 + 表示层 + 应用层
TCP和UDP的区别？
高频
特性 TCP UDP
连接 面向连接 无连接
可靠性 可靠传输 不可靠
速度 较慢 快
应用场景 HTTP、FTP、邮件 视频、DNS、游戏
TCP三次握手和四次挥手的过程？
重要

三次握手（建立连接）：

  1. 客户端发送SYN=1, seq=x
  2. 服务端发送SYN=1, ACK=1, seq=y, ack=x+1
  3. 客户端发送ACK=1, seq=x+1, ack=y+1

四次挥手（断开连接）：

  1. 客户端发送FIN=1, seq=u
  2. 服务端发送ACK=1, seq=v, ack=u+1
  3. 服务端发送FIN=1, ACK=1, seq=w, ack=u+1
  4. 客户端发送ACK=1, seq=u+1, ack=w+1

为什么四次？因为服务端可能还有数据要发送，ACK和FIN不能合并

TCP的滑动窗口和拥塞控制？

滑动窗口：实现流量控制，接收方告知发送方自己的接收窗口大小，发送方据此调整发送速率

拥塞控制算法：

  • 慢启动：cwnd从1开始指数增长
  • 拥塞避免：cwnd达到阈值后线性增长
  • 快重传：收到3个重复ACK立即重传
  • 快恢复：cwnd减半后线性增长
TIME_WAIT状态的作用？

作用：

  1. 保证最后一个ACK能被服务端收到（超时重传）
  2. 防止旧连接的延迟数据包被新连接接收

持续时间：2MSL（Maximum Segment Lifetime，约2-4分钟）

HTTP/HTTPS

HTTP/1.0、HTTP/1.1、HTTP/2、HTTP/3的区别？
重要
  • HTTP/1.0：短连接，每次请求新建TCP连接
  • HTTP/1.1：持久连接（keep-alive）、管道化（pipelining）、缓存控制
  • HTTP/2：二进制分帧、多路复用、头部压缩（HPACK）、服务器推送
  • HTTP/3：基于QUIC（UDP），解决队头阻塞，0-RTT握手
HTTPS的握手过程？
高频
  1. 客户端发送Client Hello（支持的TLS版本、加密套件、随机数）
  2. 服务端发送Server Hello（选定TLS版本、加密套件、随机数）
  3. 服务端发送证书（包含公钥）
  4. 客户端验证证书，生成预主密钥，用公钥加密发送
  5. 双方根据随机数和预主密钥生成会话密钥
  6. Finished消息，握手完成，后续使用对称加密通信
GET和POST的区别？
高频
特性 GET POST
数据位置 URL参数 请求体
幂等性 幂等 非幂等
缓存 可被缓存 不被缓存
长度限制 受URL长度限制 无限制
HTTP状态码有哪些？

1xx（信息）：100 Continue

2xx（成功）：

  • 200 OK
  • 201 Created
  • 204 No Content

3xx（重定向）：

  • 301 Moved Permanently（永久重定向）
  • 302 Found（临时重定向）
  • 304 Not Modified（缓存有效）

4xx（客户端错误）：

  • 400 Bad Request
  • 401 Unauthorized
  • 403 Forbidden
  • 404 Not Found

5xx（服务端错误）：

  • 500 Internal Server Error
  • 502 Bad Gateway
  • 503 Service Unavailable

网络安全

常见的Web攻击有哪些？如何防御？
重要
  • XSS（跨站脚本）：过滤输入、转义输出、CSP策略
  • CSRF（跨站请求伪造）：CSRF Token、SameSite Cookie、验证Referer
  • SQL注入：预编译语句、参数化查询、ORM
  • DDoS：限流、CDN、WAF、流量清洗
  • 中间人攻击：HTTPS、证书固定
什么是CORS？如何解决跨域？

CORS（跨域资源共享）：浏览器的一种安全机制，限制跨域请求

解决方案：

  • 服务端设置Access-Control-Allow-Origin
  • JSONP（只支持GET）
  • 代理服务器（nginx反向代理）
  • WebSocket
  • postMessage
Cookie、Session、Token的区别？
特性 Cookie Session Token
存储位置 客户端 服务端 客户端
安全性 较低 较高 高
扩展性 一般 差（需共享） 好（无状态）
