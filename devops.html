DevOps面试题

涵盖Docker、Kubernetes、CI/CD、监控等运维核心知识

Docker

Docker和虚拟机的区别？
高频
特性 Docker 虚拟机
启动速度 秒级 分钟级
资源占用 少（MB级） 多（GB级）
隔离级别 进程级 系统级
性能 接近原生 有损耗
Docker的镜像分层是什么？
高频

联合文件系统（UnionFS）：将多个只读层叠加，最上层可写

分层优势：

  • 共享基础镜像，节省存储
  • 构建缓存，加速构建
  • 快速部署，只需传输变更层
Docker的网络模式有哪些？
  • bridge：默认，容器通过网桥通信
  • host：共享宿主机网络栈
  • none：无网络
  • container：共享另一个容器的网络
  • 自定义网络：支持DNS解析

Kubernetes

K8s的架构组件有哪些？
高频

Master节点：

  • API Server：集群入口，处理REST请求
  • etcd：分布式键值存储，保存集群状态
  • Scheduler：调度Pod到Node
  • Controller Manager：维护集群状态

Node节点：

  • kubelet：管理Pod生命周期
  • kube-proxy：网络代理和负载均衡
  • 容器运行时：Docker/containerd
Pod、Deployment、Service的区别？
  • Pod：最小部署单元，包含一个或多个容器
  • Deployment：管理Pod的副本数、滚动更新、回滚
  • Service：提供稳定的网络访问入口，负载均衡
K8s的滚动更新和回滚如何实现？

滚动更新：

kubectl set image deployment/myapp myapp=myapp:v2
# 或修改yaml后 apply

策略参数：

  • maxSurge：更新时最多可超出的Pod数
  • maxUnavailable：更新时最大不可用Pod数

回滚：

kubectl rollout history deployment/myapp  # 查看历史
kubectl rollout undo deployment/myapp     # 回滚到上一版
kubectl rollout undo deployment/myapp --to-revision=2

CI/CD

什么是CI/CD？
高频

CI（Continuous Integration）：持续集成

  • 代码提交后自动构建、测试
  • 工具：Jenkins、GitLab CI、GitHub Actions

CD（Continuous Deployment/Delivery）：持续部署/交付

  • 自动部署到测试/生产环境
  • 持续交付需手动触发，持续部署全自动
Git工作流有哪些？
  • Git Flow：master、develop、feature、release、hotfix分支
  • GitHub Flow：简单，master + feature分支，PR合并
  • GitLab Flow：结合环境分支
  • Trunk-based：主干开发，短生命周期分支

监控与日志

常用的监控方案有哪些？

指标监控：

  • Prometheus + Grafana：时序数据库，强大的查询和可视化
  • Zabbix：传统企业级监控

日志监控：

  • ELK：Elasticsearch + Logstash + Kibana
  • EFK：Elasticsearch + Fluentd + Kibana
  • Loki：轻量级，与Grafana集成

链路追踪：

  • Jaeger、Zipkin、SkyWalking
什么是可观测性（Observability）？

三大支柱：

  • Metrics（指标）：数值型数据，如CPU、内存、QPS
  • Logs（日志）：离散事件记录
  • Traces（链路）：请求在分布式系统中的流转路径

云原生

什么是云原生（Cloud Native）？
高频

CNCF定义：

  • 容器化：Docker、containerd
  • 微服务：服务拆分，独立部署
  • DevOps：开发与运维协作
  • 持续交付：自动化部署
Service Mesh是什么？

概念：服务间通信的基础设施层，对应用透明

功能：

  • 服务发现、负载均衡
  • 流量管理（灰度发布、熔断）
  • 安全（mTLS）
  • 可观测性

产品：Istio、Linkerd、Consul Connect
